## Поэтапный план подготовки к собеседованию на позицию QA Backend (разработка автотестов, ручное тестирование)

**Контекст:**  
Ваша вакансия из log3 (HRS-44487) связана с тестированием backend-продукта, автоматизацией и знанием ряда инструментов.  
В log2 — перечень технических ожиданий от стажера, подходящих как минимальный набор знаний для успешного прохождения собеседования в такой команде.

### ЭТАП 1. ОБЩИЕ ПОНЯТИЯ ТЕСТИРОВАНИЯ И QA

1. Теория тестирования (основные виды тестов: функциональное, регрессионное, интеграционное, нагрузочное).
2. Основы тест-дизайна (классы эквивалентности, граничные значения, тестовые сценарии, чек-листы).
3. Жизненный цикл бага, принципы работы с ошибками.

### ЭТАП 2. ЗНАНИЯ ПО БАЗОВЫМ ТЕХНОЛОГИЯМ

1. Логирование (как искать и анализировать логи; примеры популярных систем: ELK, Sentry, Graylog и просто лог-файлы).
2. CI/CD:  
   - Что это, зачем нужен.  
   - Примеры инструментов: GitLab CI, Jenkins, GitHub Actions.
3. Контейнеризация (Docker — пример, взаимодействие с контейнерами).
4. Брокеры сообщений (Kafka, RabbitMQ — зачем нужны, примеры использования).
5. База данных (SQL/NoSQL, основные понятия, примеры запросов в PostgreSQL/MySQL).
6. Redis — зачем, базовое использование.
7. GIT — базовые команды (clone, commit, push, branch, merge).

### ЭТАП 3. ОСНОВЫ BACKEND, API/ТЕСТИРОВАНИЕ

1. HTTP/HTTPS — структуру, чем отличаются, какие бывают методы.
2. API:  
   - Основные типы: REST, SOAP, GraphQL, socket.  
   - Как тестировать (Postman, curl, autotests).
3. ORM — что это, зачем нужен.
4. Фреймворки для backend (Flask/Django/FastAPI — базовое представление).
5. Автоматизация тестирования API (пример: через Pytest + requests).

1. Базовые конструкции Python — типы данных, ветвления, функции, классы.
2. Исключения, логика обработки ошибок.
3. Пакетные менеджеры (pip, poetry).
4. Работа с коллекциями, comprehensions, генераторы, декораторы.
